Manual: Despliegue de un Sitio Web con Apache y MySQL en Ubuntu Server

Elaboró: M.C.C. José Adrian García López

Resumen

Este manual proporciona instrucciones para configurar una dirección IP estática en Ubuntu Server. También, proporciona una guía paso a paso para instalar y configurar un servidor web con MySQL Server y phpMyAdmin para desplegar una aplicación web.

Configuración de una dirección IP estática en Ubuntu Server

Establecer una dirección IP estática para un servidor web garantiza que la dirección IP del servidor no cambiará. Esto es fundamental para la accesibilidad del servidor web, ya que los clientes necesitan una dirección IP constante para acceder al servidor. Además, una dirección IP estática facilita la configuración de DNS y la administración de la red. A continuación, se muestra los pasos para establecer una dirección IP estática en Ubuntu Server 22.04.4 LTS.

  1. Identificar el nombre de tu interfaz de red
    Se ejecuta el comando ip a. El nombre de la interfaz suele ser algo como enp0s3 o eth0.

  2. Abrir el archivo de configuración de red
    Ubuntu Server 18.04 y versiones posteriores utilizan Netplan para la configuración de red. Los archivos de configuración de Netplan están en el directorio /etc/netplan/ y tienen la extensión .yaml. Se puede abrir el archivo con un editor de texto como nano: 

    sudo nano /etc/netplan/00-installer-config.yaml

  3. Editar el archivo de configuración
    Aquí es donde se establece la dirección IP estática, la máscara de subred, la puerta de enlace y los servidores DNS. La configuración debería tener un aspecto similar a este:

    network:
  ethernets:
    enp0s3:
      dhcp4: no
      addresses: [192.168.1.100/24]
      gateway4: 192.168.1.1
      nameservers:
        addresses: [192.168.1.100, 8.8.4.4]
  version: 2

    Es importante notar que la dirección IP 192.168.1.100 es la que se debe cambiar, seguida de su /24 que representa la máscara de subred.

  4. Guardar el archivo
    Presionar la combinación de teclas Ctrl + O, luego presiona Enter para confirmar que quieres guardar los cambios.

  5. Cerrar el archivo
    Presionar la combinación Ctrl + X.

  6. Probar los cambios
    Es una buena práctica probar la nueva configuración con sudo netplan try para verificar que la nueva configuración no contiene errores antes de aplicarla. Si la configuración fue exitosa, se debe de aplicar los cambios con el comando sudo netplan apply.

  7. Verificar la nueva configuración
    Ejecutar el comando ip a. Se debe mostrar dirección IP estática en la salida de este comando.

Instalación y configuración del servidor

Apache es una excelente opción para servir páginas PHP debido a su integración con el módulo mod_php, que permite interpretar y servir páginas PHP de manera eficiente. Además, Apache ofrece características como la reescritura de URL (a través del módulo mod_rewrite), que es ampliamente utilizada en aplicaciones PHP. Paa lograr la correcta instalación se debe de seguir los siguientes pasos.

  1. Actualizar el sistema
    Los comandos update y upgrade se utilizan para actualizar la lista de paquetes disponibles y sus versiones en el sistema operativo Ubuntu para asegurar que se está instalando la última versión de cualquier software y tener las últimas correcciones de seguridad.

    sudo apt update
sudo apt upgrade -y

    El -y es una opción que permite a apt proceder con la actualización sin pedir confirmación al usuario. apt es un manejador de dependencias de paquetes.

  2. Instalar el servidor web Apache

Antes de iniciar la instalacion de Apache, es importante mencionar que el comando sudo apt install es la base para instalar cualquier paquete de software en el sistema operativo Ubuntu Server.

Después de saber esto, para instalar Apache, que es un servidor web de código abierto muy popular por su robustez y soporte para una amplia gama de módulos adicionales, incluyendo PHP, se debe ejecutar el siguiente comando:

sudo apt install apache2

Para verificar que Apache está activo y funcionando correctamente después de la instalación, puedes utilizar el comando systemctl para comprobar el estado del servicio Apache. El comando completo sería:

sudo systemctl status apache2

Este comando te mostrará el estado actual del servicio Apache. Si Apache está funcionando correctamente, deberías ver active (running) en la salida del comando. Para salir del resumen, debe de presionar la tecla Q.

Además, se puede probar Apache abriendo un navegador web cliente conectado a la misma red y navegando a http://192.168.1.100. Esta es la dirección IP que se estableció estáticamente en la sección anterior. Si Apache está funcionando correctamente, deberías ver la página por defecto de Apache.

Instalar el MYSQL Server

MySQL es un sistema de gestión de bases de datos relacional de código abierto que es ampliamente usado en aplicaciones web, incluyendo sitios PHP. Proporciona una forma eficiente y segura de almacenar, organizar y recuperar datos, lo que es esencial para cualquier aplicación web. Para su instalación se debe de seguir los siguientes pasos:

  1. Instalar MySQL
    El primer paso es instalar MySQL en tu servidor. Esto se hace con el comando:

    sudo apt install mysql-server

    Durante la instalación, pide establecer una contraseña para el usuario root de MySQL. En este caso se establece contraseña servertso.

  2. Configurar MySQL
    Después de instalar MySQL, se debe configurar para garantizar la seguridad de la base de datos. Esto se realiza con el comando:

sudo mysql_secure_installation

Las respuestas a este paso serán de acuerdo a las preguntas, sin embargo, a la primera se responde ‘N’ y a las demás preguntas ‘Y’.

  1. Cambiar la contraseña del usuario root en MySQL
    Para establecer seguridad en la base de datos instalada, se debe cambiar la contraseña, por ende se debe entrar a mysql. Esto se realiza de la siguiente manera:

    sudo mysql

    Si al ejecutar el comando anterior marca algún error, entonces debes probar con el siguiente comando:

    sudo mysql -u root -p

    Para cambiar la contraseña debe de ejecutar el siguiente comando (reemplaza ‘tu_contraseña’ con la contraseña a establecer, para este caso fue servertso): 

    alter user 'root'@'localhost' identified with mysql_native_password by 'tu_contraseña';

    Luego, ejecuta el siguiente comando para que los cambios surtan efecto:

    FLUSH PRIVILEGES;

    Finalmente, para salir de la consola de MySQL se ejecuta el comando exit.

Instalar PHP

PHP es un lenguaje de programación de código abierto muy popular para el desarrollo web. Es fácil de aprender y usar, flexible, y tiene una gran comunidad de desarrolladores. PHP es parte del stack LAMP (Linux, Apache, MySQL, PHP), que es una solución popular para desplegar aplicaciones web.

La instalación de PHP, junto con los módulos necesarios para Apache y MySQL, se puede hacer en una sola línea con el siguiente comando:

sudo apt install php libapache2-mod-php php-mysql

Este comando instala PHP, el módulo PHP para Apache (libapache2-mod-php), y el módulo MySQL para PHP (php-mysql). Estos módulos permiten que PHP y Apache trabajen juntos para servir aplicaciones web, y que PHP interactúe con una base de datos MySQL.

Instalar phpMyAdmin

phpMyAdmin es una herramienta de software libre que se utiliza para la administración de MySQL a través de la web. Proporciona una interfaz gráfica de usuario para realizar una variedad de operaciones en MySQL y MariaDB. Algunas de las operaciones que puedes realizar con phpMyAdmin incluyen la gestión de bases de datos, tablas, columnas, relaciones, índices, usuarios, permisos, y mucho más.

Para realizar la instalación se debe seguir los siguientes pasos.

  1. Instalación
    Ejecutar el siguiente comando para instalar phpMyAdmin:

    sudo apt install phpmyadmin

    Durante la instalación, pedirá que elijas el servidor web a configurar automáticamente para phpMyAdmin. Selecciona apache2 pesionando la tecla espace, después se debe de configurar una base de datos para phpMyAdmin con dbconfig-common; selecciona “Sí” y establece una contraseña para phpMyAdmin, paa este caso se establece la contraseña servertso.

  2. Configurar Apache para usar phpMyAdmin
    Para que Apache pueda trabajar con phpMyAdmin, necesitas habilitar el módulo rewrite de Apache y reiniciar el servicio Apache. Esto se puede hacer con los siguientes comandos:

    sudo a2enmod rewrite
sudo systemctl restart apache2

    El módulo rewrite permite a Apache reescribir URLs, lo cual es necesario para algunas funcionalidades de phpMyAdmin. Reiniciar el servicio Apache asegura que los cambios surtan efecto.

Prueba del sevidor web y MySQL

Para probar el servidor web y MySQL, se clona un repositorio de GitHub y se despliega en el servidor. En este caso, se clona el repositorio alumnos: 

sudo git clone https://github.com/garcialopez/alumnos

Este comando clona el repositorio alumnos en el directorio actual. Para comprobar que el directorio se ha clonado se ejecuta un ls y se visualizan los directorios y archivos que hay.

Una vez que el repositorio se ha clonado, se debe mover los archivos al directorio del servidor web. El directorio del servidor web Apache por defecto es /var/www/html. Para mover los archivos del repositorio clonado al directorio con el siguiente comando: 

sudo mv alumnos /var/www/html/

Antes de cambiar la propiedad de los archivos, se debe navegar al directorio donde se encuentran.

cd /var/www/html/

Es importante cambiar la propiedad de los archivos a www-data, que es el usuario bajo el cual se ejecuta el servidor web Apache en la mayoría de los sistemas Linux. Esto se puede hacer con el comando chown.

sudo chown -R www-data alumnos

Este comando cambia la propiedad de todos los archivos y directorios dentro de alumnos al usuario www-data y al grupo www-data. Esto asegura que Apache tenga los permisos necesarios para leer y servir los archivos de la aplicación web.

Configuración del sitio web alumnos

El archivo database.php que se encuentra en el directorio alumnos contiene la configuración de la base de datos para la aplicación web alumnos. Se necesita abrir el archivo y modificar la configuración de la base de datos para que coincida.

Primero, se navega al directorio alumnos y abre el archivo database.php con el editor de texto nano: 

cd alumnos
sudo nano database.php

Busca la línea que comienza con $mysqli = new mysqli('localhost', 'root', 'pass', 'base de datos');.

Debes cambiar 'pass' por la contraseña de tu base de datos, en este caso 'servertso', y 'base de datos' por el nombre de tu base de datos, en este caso 'test'. La línea modificada debería verse así:

$mysqli = new mysqli('localhost', 'root', 'servertso', 'test');

Guarda los cambios y cierra el archivo.

Creación de la base de datos test

  1. Abre phpMyAdmin desde un navegador web. En este caso, debes entrar desde la dirección IP estática seguida de /phpmyadmin.
192.168.1.100/phpmyadmin

  1. Inicia sesión con el usuario root y la contraseña, para este caso es servertso.

  2. Una vez en phpMyAdmin, crea una base de datos llamada test.

  3. Dentro de la base de datos test, crea una tabla o entidad llamada alumnos con los siguientes atributos:

    • id: INT, Auto Increment (AI) Primary
    • nombre: VARCHAR(50)
    • edad: INT

Despliegue de la web

Para probar el sitio web, sigue estos pasos:

  1. Abre un navegador web y escribe la dirección IP del servidor seguido de /alumnos/index.php. Por ejemplo, si la dirección IP del servidor es 192.168.1.100, entonces debes escribir 192.168.1.100/alumnos/index.php.

  2. Al entrar, deberías poder visualizar el sitio y realizar operaciones CRUD (Crear, Leer, Actualizar, Borrar) en la entidad alumnos.
